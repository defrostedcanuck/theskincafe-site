The email-signup custom event is intentionally deferred to Phase 2's form implementation — this plan only handles what exists today.

Output: A reusable `<BookButton>` client component, 7 booking anchor replacements across 4 existing files, and `<Analytics />` mounted in `src/app/layout.tsx`.

Create `src/app/components/BookButton.tsx`:

```tsx
'use client';

import type { ReactNode } from 'react';
import { track } from '@vercel/analytics';
import { LOCATIONS } from '../lib/locations';

export type BookLocation = 'gilbert' | 'scottsdale';
export type BookSurface = 'nav-desktop' | 'nav-mobile' | 'hero' | 'locations-card' | 'booking-cta';

const URL_BY_LOCATION: Record<BookLocation, string> = Object.fromEntries(
  LOCATIONS.map((l) => [l.slug, l.bookingUrl])
) as Record<BookLocation, string>;

type Props = {
  location: BookLocation;
  surface: BookSurface;
  className?: string;
  children: ReactNode;
  ariaLabel?: string;
};

export function BookButton({ location, surface, className, children, ariaLabel }: Props) {
  return (
    <a
      href={URL_BY_LOCATION[location]}
      target="_blank"
      rel="noopener noreferrer"
      className={className}
      aria-label={ariaLabel}
      onClick={() => {
        // PII POLICY: only location + surface — NEVER email, name, message text,
        // or anything tied to an individual visitor (threat T-03-09).
        track('booking_click', { location, surface });
      }}
    >
      {children}
    </a>
  );
}
```

Hard rules:
- `'use client'` directive MUST be the first line — `track()` is client-only (Pitfall 3).
- Reuse `LOCATIONS[*].bookingUrl` from `src/app/lib/locations.ts` — do NOT hardcode Square URLs here. Single source of truth.
- `target="_blank"` with `rel="noopener noreferrer"` — the existing anchors already do this; preserve.
- `onClick` fires `track()` BEFORE navigation. The browser typically keeps the tab open long enough for the pixel POST; `@vercel/analytics` v2 Resilient Intake also handles beacon fallback.
- Surface values are a closed union — adding a new surface requires updating the type.
- `ariaLabel` is optional so existing a11y labels on individual anchors can survive the extraction.
- NEVER pass PII (`email`, `name`, free text) into `track()`. Only structural metadata.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `track()` arguments → Vercel Analytics ingestion | Event properties are sent to Vercel's analytics endpoint over HTTPS. If we ever put PII into event properties, that PII leaves our origin. |
| Booking anchor onClick → external navigation | User click opens `book.squareup.com` in a new tab. `rel="noopener noreferrer"` prevents reverse-tab-nab. |
| Client bundle → analytics script | `@vercel/analytics` injects a first-party script; cookieless by design (no consent banner required per vendor docs). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08 | Tampering (reverse tab-nab) | `BookButton` external `<a target="_blank">` | mitigate | `rel="noopener noreferrer"` baked into the `BookButton` implementation — enforced by the grep in Task 2 verify (the attribute string must appear in the file). |
| T-03-09 | Information Disclosure (PII leak to Vercel Analytics) | `track()` calls | mitigate | `track('booking_click', { location, surface })` only — schema enforced by TypeScript union types (`BookLocation`, `BookSurface`). Explicit comment in `BookButton.tsx` warns against adding PII. Task 2 verify greps for absence of obvious PII property names. |
| T-03-10 | Repudiation (missing consent for analytics) | `<Analytics />` | accept | Vercel Analytics is cookieless by design (RESEARCH.md Don't Hand-Roll table). No consent banner legally required for Vercel Analytics alone. If client later adds GA4 or Meta Pixel, a consent banner is required — that's a Phase 5+ concern. |
| T-03-11 | Denial of Service (analytics outage breaks navigation) | `BookButton` onClick | accept | `track()` is fire-and-forget; even if the analytics endpoint is down the click still navigates. Vercel Analytics v2 Resilient Intake retries quietly. |
</threat_model>
